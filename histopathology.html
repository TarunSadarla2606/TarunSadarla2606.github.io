Overview

The scaling question in clinical ML

Does data volume consistently improve performance — and what does the curve look like?

In clinical machine learning, collecting annotated data is expensive. Knowing how much labelled data you actually need to reach a given performance level matters enormously for deployment feasibility. This project addresses that question empirically: the same custom CNN architecture was trained on four progressively larger stratified subsets of the PCam dataset, with all other hyperparameters held constant. The result is a direct measurement of how ROC-AUC scales with training data volume for this task and architecture.

The full project is a 5-person team effort including a deployed web application with Django backend and React frontend. My role was the CNN architecture and the full scaling experiment — see contributions section.

Key finding: ROC-AUC consistently improves with more training data across all subsets. The improvement from 20% to 100% is real but non-linear — diminishing returns appear between 40% and 60%, which has practical implications for annotation cost planning in clinical settings.

Dataset

PatchCamelyon (PCam)

220,025 labeled 96×96 TIFF patches from digitized lymph node tissue slides. Center-cropped to 32×32 for this work. Binary classification: metastatic cancer present vs absent in the central 32×32 pixel region. Class ratio: approximately 60% non-cancerous, 40% cancerous in the raw dataset — realistic of clinical imbalance.

Architecture

Custom 5-layer CNN

5-layer convolutional network trained on 32×32 RGB patches. Binary classification via sigmoid output with BCE loss. Batch normalization and Leaky ReLU throughout, with progressive dropout schedule.

Input (32×32×3)
→ Conv2D(32) + BN + LeakyReLU → MaxPool(2×2) → Dropout(20%)
→ Conv2D(64) + BN + LeakyReLU → MaxPool(2×2) → Dropout(20%)
→ Conv2D(128) + BN + LeakyReLU → MaxPool(2×2) → Dropout(20%)
→ Conv2D(256) + BN + LeakyReLU → MaxPool(2×2) → Dropout(20%)
→ Conv2D(512) + BN + LeakyReLU → MaxPool(2×2) → Dropout(20%)
→ Flatten → FC(512) + LeakyReLU → Dropout(50%) → FC(1) + Sigmoid

# Adam (lr=0.001) · BCE loss · Early stopping (patience=5) · 30 epochs max · Batch 64

★ What this project taught — uniquely

Data volume as an empirical question, not an assumption. The non-monotonic improvement in ROC-AUC (0.81 → 0.79 → 0.81 → 0.82 across subsets) challenged the assumption that "more data always helps linearly." S2 showed surprisingly high specificity (0.931) but lower sensitivity (0.647) — the model became conservative with 40% of the data. This result illustrates that data volume interacts with model behavior in non-obvious ways, and systematic subset experiments like this are the only way to map that relationship empirically. This thinking applies directly to any clinical dataset where annotation is expensive.

Noise filtering as a prerequisite step — not an afterthought. Scanning 220,025 raw image patches with PIL image verification found 29 corrupted files. Removing them before any training — and tracking this explicitly as a pipeline step — reflects production-grade data hygiene that most academic projects skip. In clinical imaging, corrupted or mislabeled training data has patient-safety implications, making this seemingly minor step meaningfully important.

Full-stack deployment as a team collaboration challenge. This was the only project in this portfolio that combined a deep learning model with a production web application (Django backend + React frontend + REST API). Coordinating the interface between the model output (JSON with confidence scores) and the web app's display requirements required explicit API design — something that never comes up when working alone in a notebook.

PR-AUC as the honest metric under class imbalance. PR-AUC values (0.69–0.71) were consistently lower than ROC-AUC (0.79–0.82) across all subsets — and that gap is meaningful, not coincidental. ROC-AUC is optimistic under class imbalance because it accounts for true negatives (of which there are many). PR-AUC focuses on precision-recall trade-offs for the positive class (cancer), which is what actually matters clinically. Always reporting both metrics for medical imaging tasks is a lesson this project made concrete.
