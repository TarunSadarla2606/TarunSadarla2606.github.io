ASD Detection via Neuroimaging
First encounter with medical imaging AI — NIfTI brain MRI, CNN vs ViT comparison on ABIDE-I. 91% accuracy with CNN; ViT failed at ~60%. The ViT failure was the most important finding: data scale determines architecture viability.
Medical imaging pipelines · NIfTI/DICOM · Architecture trade-offs at data scale

Fetal Head Segmentation — Cine-Loop
Addressed the limitations from ASD: real clinical dataset (HC18 ultrasound), proper segmentation (Residual U-Net), and temporal modeling (3D U-Net). Built Pseudo-LDDM to generate training data when none exists — synthetic data generation as a research skill.
U-Net segmentation · 3D convolution · Synthetic data generation · Clinical measurement pipelines
Needed: Model efficiency for deployment, compression techniques
MS Research · CSCE 5934 Directed Study · Fall 2025 · Prof. Russel Pears
CNN Filter Pruning — Hybrid Crossover
Directly addressed the deployment gap: how do you get a large model to run on hospital hardware? Introduced Hybrid Crossover merging — synthesize rather than discard redundant filters. 2× compression, +0.37% accuracy, 2× faster runtime. Supervised by Prof. Russel Pears.
Model compression · Regression-based synthesis · Guard-rail engineering · Production-aware pruning
Needed: Multi-modal data, audio/signal processing, cloud-scale data engineering

Where this is all heading — one possible project as an example

Every project in this MS taught one piece: ASD taught imaging pipelines. Fetal Head taught segmentation and temporal modeling. CNN Pruning taught compression for deployment. WECARE taught edge constraints. Big Data taught cloud-scale pipelines. BirdCLEF taught feature engineering. The SDAI course taught full-stack deployment. AIWH taught clinical data structure (MIMIC-IV).

One example of a project that could tie all of this together — not yet committed to, but illustrative of the kind of work this portfolio leads toward: a chest X-ray AI system (ClinicalCXR-AI) using the MIMIC-CXR + MIMIC-IV datasets — combining a DenseNet-121 visual backbone with a clinical metadata MLP in a late-fusion architecture, deployed via FastAPI. This would use: DICOM-native image loading (from ASD + Fetal Head), late fusion architecture (from Multimodal Emotion), deployment pipeline (from WECARE + BirdCLEF), clinical data schema knowledge (from AIWH), and model compression awareness (from CNN Pruning). Each piece came from somewhere in this portfolio.

This is the kind of work I want to do full-time — and I'm available to start from June 2026.

Research Interests

What I'm thinking about

Problems I find genuinely interesting and want to work on next — driven by gaps encountered across this portfolio.

01
Explainability in Clinical AI
A model that achieves 97% on a benchmark tells a clinician nothing if they can't see why it decided what it did. Grad-CAM, SHAP, and cross-modal attention as trust mechanisms — not just academic reporting artifacts. LIME in the ASD project exposed how hard this is in practice.
02
Low-Data Medical Imaging
The ViT failure in the ASD project (60% with 100K slices) proved the point: annotation is expensive and data is scarce in clinical settings. Few-shot learning, self-supervised pretraining, and synthetic augmentation (like Pseudo-LDDM) as principled responses to that constraint.
03
Clinical Deployment Constraints
Model compression, quantization, and real-time inference for imaging pipelines — from laptop GPU to hospital workstation. WECARE's 0.033ms result and the CNN pruning 2× speedup showed this is achievable without sacrificing clinical utility.
04
Temporal & Longitudinal Imaging
Static frames lose information. Cine-loop ultrasound, longitudinal MRI follow-up, and time-series biosignal data reveal dynamics that single-image models miss entirely. The fetal head cine-loop work is one starting point; extending this to 3D volumetric imaging is the clear next step.
05
Multimodal Clinical Fusion
The fusion failure in the multimodal emotion project taught a specific lesson about what simple concatenation cannot do. Cross-modal attention and temporally-aligned fusion — now applied to imaging + clinical metadata (like MIMIC-CXR + MIMIC-IV) — is the architecture problem worth solving next.
06
Multi-Site Dataset Generalization
ABIDE-I's 17 scanner sites introduced scanner bias that training doesn't automatically resolve. Domain adaptation, federated learning, and site-specific normalization for multi-centre clinical studies — where scanner heterogeneity is the norm, not the exception.
