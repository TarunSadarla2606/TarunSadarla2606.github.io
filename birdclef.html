Overview

The problem: 152 species, noisy field recordings, no spectrograms

A deliberate choice to use structured acoustic features rather than end-to-end deep learning.

The BirdCLEF 2022 Kaggle competition presents passive audio recordings from Hawaiian bird monitoring stations — real field conditions with background noise, wind, and overlapping calls across 152 species. Most participants approach this with raw spectrogram-based deep learning. My approach instead centers on systematic acoustic feature engineering with librosa combined with hierarchical taxonomic classification — predicting taxonomic Order first, then narrowing to Family, rather than attempting direct species-level classification from tabular features.

Design rationale: Direct 152-class classification from 40–107 hand-crafted tabular features is intractable. The eBird taxonomy provides a natural hierarchy: 17 Orders contain 47 Families which contain 152 species. Classifying at Order then Family level makes the problem decomposable and interpretable — you can understand which acoustic features drive each level's decisions.

Feature Engineering

A complete audio feature extraction pipeline

All features extracted from raw .ogg audio using librosa, with a 1–8 kHz Butterworth bandpass filter applied first to isolate typical birdsong frequencies and suppress low-frequency environmental noise.

Augmented set — 107 features

The augmented set adds temporal variation and additional spectral descriptors to the base 40: standard deviations for MFCC, Chroma, and Spectral Contrast (32 features); delta-MFCC mean and std capturing temporal derivatives (26 features); Spectral Rolloff, Flatness, and RMS Energy mean + std (6 features). All features are min-max normalized to [0,1] per column post-extraction.

Geographic coordinates are encoded as Cartesian (x, y, z) on a unit sphere — this preserves circular continuity near the antimeridian, unlike raw degree values which jump discontinuously at ±180°.

Classification Pipeline

Hierarchical two-level taxonomy classification

Rather than attempting direct 152-class species prediction, the pipeline classifies at two levels of the eBird taxonomy. Each level uses its own separately trained model, with class imbalance addressed by SMOTE before fitting.

Stage 01
Feature extraction
Bandpass filter → 40 or 107 acoustic features per recording via librosa. Geographic encoding applied.
Stage 02
Level 1 — Order prediction
XGBoost / FNN predicts taxonomic Order (17 classes). SMOTE applied. Stratified 70/15/15 split.
Stage 03
Level 2 — Family prediction
XGBoost / FNN predicts Family (47 classes) from the same features. Output: Order + Family + confidence score.

Models

XGBoost (primary): Bayesian hyperparameter search via BayesSearchCV from skopt. Key hyperparameters tuned: n_estimators, max_depth, learning_rate, subsample, colsample_bytree, gamma, lambda. One-vs-All (OvA) strategy also implemented as a separate binary classifier per class at Order level.

Feedforward Neural Network (comparison): Grid search over hidden layers (2–4), hidden units (64/128/256), dropout rate (0.2/0.3). Fixed 3-layer architecture (256→128→64) also evaluated on augmented features. Early stopping (patience=10). Both models evaluated with macro F1 and balanced accuracy.

AWS deployment

The final Streamlit application was deployed globally on AWS, accepting uploaded audio recordings and returning: predicted bird Order and Family (hierarchical label), and per-class confidence scores with interactive display. Live demo video available via the repository.

★ What this project taught — uniquely

Acoustic feature engineering from scratch with librosa. Every other project in this portfolio uses image data or wearable sensor signals with well-established preprocessing conventions. This was the first time I worked with raw audio, and had to understand what MFCC, Chroma, Spectral Contrast, and Spectral Rolloff actually represent physically — not just call them as functions. Building intuition for what makes a birdsong feature discriminative versus noise-correlated was a fundamentally different kind of learning.

Hierarchical taxonomic classification design. Decomposing a 152-class problem into a two-level Order → Family hierarchy — with separate SMOTE and separate models at each level — was an architectural decision I had to reason through from scratch. It introduced me to thinking about classification not as a single flat prediction but as a structured inference problem that mirrors real-world taxonomic knowledge.

Spherical geographic encoding. Using Cartesian (x, y, z) coordinates on a unit sphere rather than raw latitude/longitude was a small detail that surprised me when I discovered why it matters — raw degree values introduce a discontinuity at ±180° that disrupts any distance-based computation. It was the first time I encountered a feature engineering problem rooted in geodesy rather than statistics.
