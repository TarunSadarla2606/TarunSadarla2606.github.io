Background & Motivation

Why smaller, faster models matter in healthcare AI

And why the obvious solution — just remove filters — isn't good enough.

🌐 For everyone

When an AI model is trained to read medical images, it ends up with thousands of internal components called filters — each one looking for a specific pattern in the image, like an edge, a texture, or a shape. The problem is that many of these filters are redundant: they're detecting nearly the same thing as another filter. This wastes memory, slows down inference, and makes it harder to deploy AI in resource-constrained environments like hospital workstations or wearable devices.

The obvious fix is to find the redundant filters and delete them. But deletion is permanent — any information encoded uniquely by the removed filter is gone. This project asks: what if instead of deleting, we synthesize? What if we create a brand-new filter that captures the best features of both redundant ones, and use that single filter to replace both?

⚙️ Technical context

This work is directly motivated by the clinical deployment problem. A model achieving 97% on a benchmark is useless if it requires a data centre to run. Structured pruning produces architecturally smaller networks — fewer channels, fewer MACs, real latency reductions — without the sparse weight matrices that unstructured pruning produces and that hardware struggles to exploit efficiently.

Where it started: Ashwini Sharma's founding observation

This project is directly rooted in prior MS thesis work at UNT. Ashwini Sharma studied gender classification on CelebA using a custom 5-layer CNN and found that only a small subset of filters in the final convolutional layer contributed meaningfully to predictions — the rest were largely redundant. That observation raised the question this directed study was built to answer.

How the methodology evolved — week by week

Week 1

From IoU to Cosine Similarity

The initial filter similarity metric — IoU on thresholded activation maps — was replaced with cosine similarity. Hierarchical clustering with medoid replacement introduced as the grouping strategy.

Week 2

The Stopping Criterion Problem

Central open problem identified: how do you decide when to stop pruning? Any fixed similarity threshold is arbitrary. Establishing a principled, data-driven stopping criterion became the key research challenge.

Week 3

Correction-Probability Breakthrough

Merging decisions grounded in a new metric: does filter B actually correct the errors that filter A makes? Moving from geometric similarity to functional contribution — a fundamentally more meaningful criterion.

Week 4

Two Tracks Emerge

The directed study diverged into two parallel approaches: Local HC (my teammate's layer-local medoid merge strategy) and Global Scoring (cross-layer ranking). This separation enabled a genuine head-to-head comparison.

Later

Hybrid Crossover Introduced

My core novel contribution: replacing the standard "drop the weaker filter" action with regression-based synthesis. ILR scoring, accuracy guard rails, and the full Global pipeline built and tested against hard accuracy constraints across both architectures and both datasets.

My Novel Contribution

The Hybrid Crossover: synthesize, don't discard

The core idea I introduced and implemented for this directed study.

🌐 For everyone

Standard pruning works like this: find two filters that do basically the same thing, keep the better one, throw away the other. Simple — but permanent. The unique information encoded in the discarded filter is lost forever.

My approach works differently. Instead of throwing one filter away, I create a brand new filter whose job is to capture the most important activations from both parents. Think of it as creating a "best of both worlds" filter through mathematical optimization, then replacing both originals with this single synthesized one. Same compression (two become one), but far less information loss.

⚙️ Technical detail

Given two candidate filters A and B identified as redundant by the Global scoring pipeline:

Step 01
Feature Map Generation
Run both filters on calibration data → activation maps FM_A and FM_B
Step 02
Hybrid Target Synthesis
Target[i,j] = max(FM_A[i,j], FM_B[i,j]) — preserves peak activations from both parents
Step 03 ★
Weight Regression
F_new = argmin_W ‖Conv(X,W) − Target‖² — solved via gradient descent
Step 04
Replacement
Replace both filters A and B with the single synthesized F_new
# Hybrid target: preserve peak activations from both parents
target_hybrid = torch.max(fm_a, fm_b) # element-wise max

# Solve for new filter weights via gradient descent
f_new = nn.Parameter(torch.randn_like(filter_a))
optimizer = torch.optim.Adam([f_new], lr=1e-3)
for _ in range(n_steps):
    pred = F.conv2d(calibration_data, f_new)
    loss = F.mse_loss(pred, target_hybrid)
    loss.backward(); optimizer.step()

# Replace both original filters with synthesized one
layer.weight.data[idx_a] = f_new.data
layer.weight.data = delete_channel(layer.weight.data, idx_b)

Why Hybrid Crossover is faster, not slower: Standard drop, when applied aggressively, frequently violates accuracy guard rails and triggers rollback — the algorithm undoes the change, soft-locks the channel, and searches for an alternative. This is expensive. The Hybrid Crossover avoids this because the synthesized filter approximates the pre-pruning representation by construction. Fewer rollbacks → 2× faster overall runtime despite the additional regression step.

The Global Scoring pipeline it integrates with

🌐 For everyone

The Hybrid Crossover doesn't work in isolation — it's the merging strategy inside a larger system that decides which filters to merge in the first place. That system is the Global Scoring pipeline, which ranks all filters across the entire network simultaneously and prunes the least important ones first.

⚙️ Technical detail

Filters are scored using ILR (Inside-Layer Ranking) — a weighted fusion of three signals computed from forward passes only, making it fast, stable, and reproducible across long runs:

Activation RMS
0.6
Post-ReLU activation magnitude — high weight because directly measures filter contribution to downstream activations
BN-γ Magnitude
0.4
Batch Norm scale parameter — reflects how much the network has learned to use this channel
HRank / Frobenius
0.2
Feature map energy — lower Frobenius norm indicates low-rank, less informative filters

Each signal is per-layer min-max normalized to [0,1], then fused into a single importance score. Filters are ranked globally (across all layers at once), not layer by layer. This cross-layer ranking is what allows the Global approach to concentrate pruning on the most compute-intensive blocks — which is why it outperforms Local HC on both compression ratio and latency speedup.

Accuracy guard rails: the safety mechanism

🌐 For everyone

One of the most important engineering decisions in this work: we never let the model's accuracy drop too far. After every structural change, we check — and if we've gone too far, we undo it. Think of it as an undo button with a defined tolerance budget.

# After every structural change (prune or merge):
if (baseline_acc - current_acc) > Δ_max_overall: ROLLBACK
if (baseline_class_acc[c] - current_class_acc[c]) > Δ_max_class: ROLLBACK

# CelebA limits
Δ_max_overall = 2 pp  |  Δ_max_class = 6 pp

# CIFAR-10 limits
Δ_max_overall = 3 pp  |  Δ_max_class = 9 pp

The Global approach adds per-step limits on top of cumulative limits, and uses soft locks to temporarily exclude channels that repeatedly trigger rollbacks — preventing the optimizer from spinning in place.

Results

Head-to-head: Hybrid Crossover vs. Standard Drop

Same architecture, same dataset, identical guard rails. The numbers speak clearly.

🌐 For everyone

We ran both approaches — my Hybrid Crossover and my teammate's Standard Drop — on the exact same model (VGG-16 with Batch Normalization) and the same dataset (CelebA gender classification), with identical accuracy limits. Every other variable was held constant. The results show that synthesizing is better than discarding on every metric that matters.

The counterintuitive result: The Hybrid Crossover is both more aggressive (2× more channels removed) AND more accurate AND faster. This is because regression-synthesized filters satisfy guard rails on the first or second attempt — standard drop under aggressive compression triggers expensive rollback loops that dominate total runtime.

Limitations & Future Work

What remains to be done

Current limitations

Middle layers (B3, B2) resist compression due to low filter redundancy — the Hybrid Crossover, like Standard Drop, finds few valid merge candidates there. This limits overall compression ratios on deeper architectures.

All experiments are on binary classification tasks (CelebA gender, CIFAR-10 truck/ship). Multi-class extension and medical imaging domain validation remain open.

The Hybrid Crossover has only been tested within the Global framework — applying it within Local HC is unexplored.

Next steps

Phase 4 fine-tuning — short retraining after pruning to recover accuracy in middle layers, which currently resist compression due to low redundancy between filters.

Multi-class and medical imaging — extend evaluation to ImageNet-scale classification and histopathology patch classification, where filter redundancy patterns may differ substantially from binary tasks.

INT8 quantization — combine structured pruning with quantization for embedded deployment, targeting the intersection of model size and inference precision requirements.

Quantile-adaptive threshold — replace the static cosine similarity cutoff with a principled per-layer quantile, adapting to actual filter redundancy distributions rather than using a global fixed value.

★ What this project taught — uniquely

Synthesis over deletion as a compression philosophy. Every other model compression paper in the literature treats structured pruning as removal: find a redundant filter, remove it, accept the information loss. The Hybrid Crossover inverts this: when two filters are redundant, synthesize a new one that preserves both their salient activations. This reframing — from a removal problem to a synthesis problem — is the conceptual novelty of this work, and it produced empirically better results. It directly applies to any compression task where information retention matters more than maximum removal.

Guard-rail engineering as responsible compression. Most pruning papers report accuracy after pruning but say nothing about what accuracy drop was permissible during the process. The hard guard rails (≤2pp overall, ≤6pp per-class on CelebA) enforced clinical-grade reliability constraints during compression — every structural change was validated before being committed, with rollback if violated. This is the kind of engineering discipline that separates research-grade from deployment-grade work.

Why the faster method is also the better method. The Hybrid Crossover is 2× faster than Standard Drop not despite being better, but because of it. Regression-synthesized filters satisfy accuracy constraints on the first or second attempt; Standard Drop under aggressive compression triggers expensive rollback loops. Understanding that accuracy-preserving design choices can also reduce compute cost — that quality and efficiency are not always in tension — is a useful heuristic for any optimization problem.

ILR over Taylor saliency for production stability. Taylor-based saliency requires gradient computation through the model, which is sensitive to training mode, mixed-precision settings, and compiler behavior. ILR uses only forward-pass statistics (activation RMS, BN-γ, HRank) — cheaper, reproducible, and stable across long runs with many architectural changes. Learning to choose a simpler, more stable signal over a theoretically richer but practically fragile one is a production engineering skill.
