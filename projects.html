The clinical problem: ASD diagnosis typically takes 18–24 months from referral to confirmed diagnosis — limited by the availability of specialist behavioural assessment time. Structural MRI is already routinely acquired in paediatric neurodevelopment referrals. This system analyses that existing data algorithmically, flagging high-probability cases for expedited specialist review.

In clinical terms: 95.6% sensitivity means the system correctly identifies 956 out of every 1,000 children who have ASD. 97.2% specificity means it raises a false alarm in only 28 out of every 1,000 children who do not. The probability estimate is well-calibrated — when the model says 85% likely, it is genuinely correct about 85% of the time, which is what makes the uncertainty flag meaningful.

Site variance is the critical finding: Performance spans 88.5% to 98.5% sensitivity depending on the scanner used — a 10-point gap. In a production deployment, site-specific calibration would be required. The system displays a site-specific reliability badge on every output and documents this in the Model Card.

What was rebuilt post-course: The 2023 B.Tech baseline had no explainability, no uncertainty, and no clinical output. The 2026 rebuild added GradCAM + LIME dual spatial explanation, MC-Dropout uncertainty (30 stochastic passes), PDF clinical report with LLM narrative, site reliability indicators, and a Model Card with FDA SaMD Class II framing.

Clinical impact
Pre-assessment triage layer — identifies high-probability cases for expedited specialist review, targeting the 18–24 month diagnostic delay
What a clinician receives
P(ASD) probability with confidence interval, spatial heatmap of brain regions driving the inference, uncertainty flag, site reliability indicator, PDF clinical report
Regulatory status
Research-grade decision support only. FDA SaMD Class II / De Novo pathway documented. Known failure modes in Model Card.
What the 2023→2026 gap taught
A classifier with 99.4% AUC that provides no explanation and no uncertainty estimate is not a clinical tool. That gap became the architecture of everything built after.

→ What the ASD project couldn't do

The ASD system classified from whole-brain structural MRI, but it measured nothing. Clinical practice often needs a precise numerical measurement — not just a binary prediction. The fetal head project moved from classification to clinical measurement, adding a temporal dimension (cine-loop sequences) and a deployment constraint (the model needs to run in a busy ultrasound suite, not a research server). The CNN pruning work runs in parallel, directly motivated by the question: how do you get the segmentation model small enough to actually deploy?

The clinical problem: HC measurement is mandatory at every routine antenatal scan (18–22 weeks, 28–36 weeks). Manual calliper placement takes 2–4 minutes, introduces up to 7mm inter-observer variation, and accumulates to approximately 153 sonographer hours per year at a unit performing 20 scans per day. This system replaces that manual step with a measurement that is reproducible to 1.75mm — more than 40% inside the ISUOG clinical acceptability threshold of ±3mm.

In clinical terms — what 97.36% Dice means: The AI draws the skull boundary on an ultrasound image. 97.36% Dice means that, on average, 97.36% of the area the AI labels as "skull" genuinely is skull, and 97.36% of the actual skull area is correctly identified. At 1.75mm HC error, the measurement is within the range of normal day-to-day variability between experienced sonographers.

The cine-loop system: A sonographer doesn't assess HC from a single frozen frame — they sweep the probe and select the clearest cross-section from a short clip. The temporal model processes 16-frame sequences, identifying which frames had the clearest skull interface and producing a consensus measurement with a frame-level uncertainty map showing where the boundary was ambiguous. Measurement error: 2.10mm — still within the ISUOG threshold.

The pruning connection: A segmentation model that achieves 97% Dice on a research GPU is not clinically useful if it cannot run on hospital hardware within workflow timing constraints. The CNN pruning work (Hybrid Crossover method) delivered 2× compression with +0.37% accuracy improvement on VGG-16 — a technique directly applicable to deploying the segmentation backbone in resource-constrained clinical environments. The pruned model retains all clinical accuracy while halving memory footprint and doubling inference speed.

Clinical impact
153 sonographer hours saved per unit per year. Reproducible to 1.75mm — eliminates calliper-placement variability. Within ISUOG ±3mm threshold on both static and cine modes.
What a clinician receives
HC in mm, gestational age ± 2-week CI (Hadlock 1984), trimester classification, GradCAM++ skull boundary overlay, frame-level uncertainty map, dual-mode PDF report
Pruning for deployment
Hybrid Crossover: synthesises a new filter from two redundant ones instead of deleting. 2× compression, +0.37% accuracy, 2× faster inference. Enables deployment on clinical hardware without accuracy penalty.
Known limitation
Third-trimester accuracy (MAE 7.60mm) exceeds ISUOG threshold — acoustic shadowing from the ossified skull is documented in Model Card with recommendation for manual verification >>30 weeks GA.

→ What the fetal head project exposed

The fetal head system demonstrated that the inference pipeline could be clinically accurate — but it was deployed to HuggingFace Spaces, not to a hospital. Real clinical deployment requires a local-first system, a production REST API, and an interface a clinician can use without a data science background. The histopathology project was where those capabilities were built: Django REST API, React frontend, and the first time a clinical AI system in this portfolio could run on a clinician's own infrastructure, independent of cloud services.

The clinical problem: Digital pathology labs are processing thousands of whole-slide images daily — far beyond the capacity of manual review alone. Automated cancer detection pipelines need to be not only accurate but deployable locally, with a clinical interface that pathologists can integrate into their existing review workflow without sending data to the cloud.

In clinical terms — what AUC-ROC 0.82 means: The model correctly ranks a cancerous patch above a non-cancerous patch 82% of the time when given one of each at random. In a screening pipeline where the goal is to shortlist cases for pathologist review rather than make autonomous diagnoses, this discrimination performance is useful as a first-pass filter.

The data scaling finding: The 4-stage volume study showed non-monotonic performance — adding more training data past a certain threshold initially degraded performance before recovering. This is a clinically relevant result: it means that "collect more data" is not always the right answer for a screening pipeline, and that validation at multiple data scales should be a standard part of clinical AI development.

The deployment capability this project built: Django REST API with a React frontend, deployable locally without cloud infrastructure. Every clinical AI project after this one inherited the ability to produce a clinical interface that runs on hospital hardware rather than external servers.

Clinical impact
First-pass screening layer for pathology review queues — ranks patches by cancer probability to surface high-priority cases for pathologist attention first
Key finding (operationally relevant)
Non-monotonic data scaling: more data ≠ better performance at intermediate volumes. Clinical AI validation must include multi-stage data volume testing, not just final benchmark performance.
Deployment capability acquired
Local-first Django REST + React deployment. Pathologist-facing interface independent of cloud. The template for production clinical AI integration — carried into all subsequent systems.

WECARE — Cardiac & Fall Detection
ECG F1: 0.9864 · 0.033ms Inference

On-device arrhythmia (MIT-BIH) and fall detection (MobiFall). 3-block 1D CNN, TorchScript, no cloud dependency. Threshold set at 0.65 — deliberately conservative to minimise missed falls in a population that can't easily call for help. Demonstrates clinical threshold reasoning: the right number is not the number that maximises accuracy, it's the number that minimises the clinically dangerous error type.

Model Compression
CNN Filter Pruning — Hybrid Crossover
2× Compression · +0.37% Accuracy · 2× Speed

Novel structured pruning method: instead of discarding redundant filters, synthesises a new filter from two via regression. VGG-16 backbone. Direct motivation: the fetal head segmentation model needs to run on clinical hardware. 2× compression with accuracy improvement — not just preservation — under hard guard-rail constraints. The Hybrid Crossover method is the research contribution; Standard Drop is the comparison baseline.
