Project 01

Glass Identification using Classical ML

UCI Glass Dataset · RBF Kernel · KNN · Logistic Regression · MLP

Multi-class classification of 6 glass types from 9 chemical composition features (RI, Na, Mg, Al, Si, K, Ca, Ba, Fe) on the UCI Glass dataset (214 samples). The central experiment is a systematic study of how RBF kernel transformation affects class separability and model performance — comparing the same classifiers before and after kernel mapping.

Methods

  • ▸MinMax normalization of all 9 chemical features to [0,1]
  • ▸t-SNE visualization before and after RBF kernel transformation
  • ▸RBF kernel mapping via rbf_kernel(γ=0.9)
  • ▸Linear Perceptron, Logistic Regression, MLP — all before and after kernel
  • ▸Feature importance via Logistic Regression coefficients → identified 2 least significant features
  • ▸RepeatedKFold + GridSearchCV for hyperparameter tuning

Tarun's contributions

  • ▸KNN and Perceptron model implementation and evaluation
  • ▸RBF kernel transformation pipeline
  • ▸t-SNE visualization before and after kernel mapping
  • ▸Feature importance analysis and feature selection experiments

★ What this project taught — uniquely

Kernel methods and the kernel trick. This was the first time I worked directly with kernel functions as explicit feature map transforms — computing the RBF kernel matrix, using it as input features, and directly observing via t-SNE how kernel mapping restructures the feature space. Understanding that you can implicitly project into infinite-dimensional spaces without computing the projection explicitly (the kernel trick) became concrete when I saw it working — or not working — on the glass dataset's awkward 6-class distribution.

Feature importance analysis beyond permutation. Using Logistic Regression coefficients to identify and then ablate the two least significant chemical features — and measuring the actual performance change — gave me a practical view of what "feature importance" means statistically versus what it means for model performance. The two aren't always aligned.

Project 02

Image Classification on CelebA using VGG-16

162K+ images · Gender Classification · Facial Landmark Analysis · KerasTuner

Fine-tuning VGG-16 on the CelebA dataset (162,770 train / 19,962 val / 19,867 test images) for gender classification and facial landmark-based attribute analysis. The key experiment investigates whether mouth-width or eye-width quartile (derived from CelebA landmark coordinates) is predictive of smiling or gender classification accuracy.

Test accuracy: 0.9594 on gender classification with fine-tuned VGG-16. Confusion matrix: 10,844 TN · 565 FP · 242 FN · 8,216 TP.

Joint work (Tarun + Ramyasri)

  • ▸KerasTuner hyperparameter search — neurons (256/512), dropout (0.3/0.4/0.5), optimizer (Adam/SGD)
  • ▸Best config: Dense(256), Dropout(0.5), Adam · 30 epochs final training
  • ▸Data generator class for efficient batching of large-scale image data

Tarun's individual contributions

  • ▸R1 — Fine-tuned VGG-16 for binary gender classification; test accuracy 0.9594
  • ▸R3a — Mouth width index computed from landmark coordinates; Q1 vs Q4 quartile split; VGG-16 classifiers for smiling per quartile
  • ▸R3b — Precision-based conclusion on whether mouth width predicts smiling status

★ What this project taught — uniquely

Systematic hyperparameter search with KerasTuner. All other projects in this portfolio use manual or grid hyperparameter tuning. This was the first use of automated search (KerasTuner) on a large-scale image dataset — learning how to define a search space, run trials efficiently, and interpret search results without overfitting to the validation set during the search process itself.

Facial landmark analysis as a continuous variable. Computing mouth width from raw CelebA landmark pixel coordinates, converting it to a continuous index, then dividing into quartiles to test whether the quartile distribution interacts with a separate classification task (smiling) — this multi-variable experimental design was new. It's a way of asking whether a proxy feature (face geometry) contains information that overlaps with a labeled attribute.

Project 03

Bayesian Network — Car Won't Start

Probabilistic Reasoning · DAG · Conditional Probability Tables · networkx

Construction, visualization, and probabilistic querying of a DAG-based Bayesian Network representing a car fault diagnosis scenario. Eleven nodes representing causal dependencies between battery age, alternator, fanbelt, oil level, and observable symptoms. The project demonstrates probabilistic inference through marginalization over intermediate variables.

Network structure

battery age → battery dead → battery flat → {lights, gas gauge, car won't start} · alternator broken → no charging → battery flat · fanbelt broken → no charging · battery flat + no oil → car won't start · no oil → dipstick

Probability queries implemented

RequirementQueryResult
R1Build and visualize DAG via networkx11-node DAG with all CPTs
R2P(car won't start | +battery age)Computed by marginalization
R3P(+lights | +battery age)Conditional propagation
R4P(+no charging | +alternator broken, +fanbelt broken)Multi-evidence inference

★ What this project taught — uniquely

Probabilistic inference through marginalization by hand. Every other project in this portfolio uses gradient-based optimization or statistical estimators. This was the only project requiring manual computation of conditional probability distributions through summing over intermediate variables — implementing the chain rule of probability, not calling a library to do it. Writing out each marginalization step made the relationship between graph structure and inference cost concrete in a way that reading about it never did.

DAG-based causal structure. Bayesian networks represent conditional independence assumptions, not just correlations. Building a DAG where causal direction matters — battery age causes battery dead, not the reverse — and then verifying that the network's probability tables encode the right dependencies, was a fundamentally different kind of modelling than anything else in this portfolio.
