The Problem

Why measuring a baby's head is harder than it sounds

And why AI has been solving only half of it.

🌐 For everyone

During pregnancy, doctors regularly measure the size of the baby's head using ultrasound — a technique that uses sound waves to create images of the inside of the body. This measurement, called Head Circumference (HC), tells them how the baby is growing and flags developmental problems early. The standard method is a sonographer manually tracing an ellipse around the head on a frozen 2D image. It works, but it is slow, subjective, and the result changes depending on who does it and which frame they happen to pick.

⚙️ Technical detail

Deep learning approaches have largely solved single-frame segmentation — models achieve >97% Dice on the HC18 benchmark. But clinical ultrasound is acquired as a cine-loop — a continuous video sequence of 30+ seconds — not a static snapshot. Applying a frame-wise model to cine data produces flickering, temporally unstable contours that erode clinical trust even when individual frame accuracy is high. The community has largely ignored this temporal dimension.

The core gap we addressed: No public cine-loop fetal ultrasound dataset exists at scale. The HC18 challenge — the field's standard benchmark — contains 1,334 static images with no temporal sequences. Building a temporally-aware model requires solving a data problem before the modelling problem.

Clinical challenges

Ultrasound images are inherently noisy — speckle interference, acoustic shadowing behind bone, and motion artefacts all obscure the skull boundary in ways that shift unpredictably frame to frame. The fetal head also occupies a small, variable region of the frame as the probe moves.

What we built

A two-part pipeline: a strong static-frame baseline using a Residual U-Net, and a temporally-aware system using a 3D U-Net trained on synthetic video sequences we generated ourselves — complete with a per-video reliability score so clinicians know how trustworthy each prediction is.

Approach — Part 1

Baseline: Teaching the AI to segment from a single frame

My work — full implementation from raw images to final measurements.

🌐 For everyone

Think of this as training an AI to draw an accurate outline around the baby's skull in a single ultrasound image. We tested four different ways of preparing the images before showing them to the model, ranging from minimal cleaning (RAW) to heavy processing (skeletonized), to find out which preparation gave the best results.

⚙️ Technical detail

I implemented and trained a Residual U-Net across four input variants representing different preprocessing depths. Training used BCE + Dice hybrid loss, Adam optimiser with mixed-precision via LossScaleOptimizer, on a 75/20/5 train/validation/test split of HC18. Post-processing applied morphological closing, hole-filling, largest-component selection, and Ramanujan-based ellipse fitting for HC computation in millimetres. 32.4M parameters, input resolution 384×256.

Figure 1 — Full baseline pipeline: HC18 dataset → 4 preprocessing variants → Residual U-Net → post-processing → ellipse fitting → HC measurement in mm

Preprocessing: four ways to prepare an ultrasound image

🌐 For everyone

Before the AI sees an image, we run it through a series of filters — noise reduction, edge sharpening, clustering — similar to adjusting brightness and contrast on a photo before printing it. We tried four different levels of this preparation to see how much cleaning actually helps the model.

⚙️ Technical detail

Pipeline: median filter (kernel=5) → Gaussian filter (7×7, σ=1.0) → Wiener filter (window=7×7) → Sobel edge detection → K-means clustering (k=3) → morphological cleaning → skeletonization. Four variants — RAW (minimal), SEG (standard segmentation pipeline), IMP (improved with extra filtering), SKL (fully skeletonized to thin boundary lines).

The model: Residual U-Net architecture

🌐 For everyone

The model we used is a type of neural network called a U-Net — named for its U-shape, where it first compresses the image to understand what is in it, then expands back out to draw a precise boundary. The "Residual" part adds shortcut connections that help it learn more reliably without losing information during training.

Approach — Part 2

Pseudo-LDDM: Generating video data from still images

My teammate's contribution — simulating realistic ultrasound video without a single real video clip.

🌐 For everyone

The biggest challenge was that no public ultrasound video dataset exists for fetal head measurement. Rather than abandon the video approach, we built a system that generates convincing synthetic video from the 999 still images we did have. Think of it as a physics-based animation engine: it understands how an ultrasound probe moves, how soft tissue deforms around rigid bone, how shadows fall, and how brightness shifts with depth — producing realistic fake video from a single frozen image.

⚙️ Technical detail

The Pseudo-LDDM was built progressively in four phases, each adding a layer of anatomical and clinical realism. Each output is a 16-frame clip saved as a compressed .npz file with aligned mask arrays, ready for supervised training of the 3D U-Net. This work was led and implemented entirely by my teammate Ramyasri Murugesan.

Phase 01

Basic Motion — Proof of Concept

The first version: simple back-and-forth movement applied to the image, like gently rocking the camera. Proves the video generation pipeline works, but the motion looks mechanical and has no understanding of anatomy.

Affine transformations (rotation + translation) via sinusoidal functions · Gaussian noise added per frame · No anatomical constraints yet

Phase 02

Anatomical Constraints

Now the simulation knows which part of the image is bone (the skull) and which is soft tissue. Bone stays rigid and fixed; only soft tissue deforms. The motion starts to look medically realistic.

Motion split: rigid region (skull mask) + non-rigid region (soft tissue via cv2.remap optical flow) · Rician speckle noise model introduced

Phase 03

Temporal Realism — ~30 Second Sequences

Longer, more natural motion. The probe drifts unpredictably rather than repeating the same cycle, and occasional bursts simulate the baby moving. No two generated sequences look the same.

Sinusoidal motion replaced by cumulative random walk for long-term probe drift · Sporadic Gaussian spikes injected to simulate fetal movement events · Non-cyclical, non-repeating long-form sequences

Phase 04 ★

Clinical Fidelity — Full Realism

The final version adds the subtle imperfections of real clinical scanning: the acoustic shadow behind the skull, the tiny tremor of the operator's hand, brightness changes with depth, and unique motion bursts. This is what actual clinical ultrasound video looks like.

Dynamic acoustic shadowing behind skull mask · High-frequency probe jitter simulating operator tremor · TGC intensity drift · Unique non-rigid flow field per wiggle event

Phase 1 → Phase 4 improvements: Motion stability (TGV) improved 55% · Boundary Dice stability improved from 0.755 → 0.909 · SSIM decreased (more artefact realism — intentional) · KL Divergence increased (richer texture distribution matching real ultrasound)

3D U-Net: watching video, not just frames

🌐 For everyone

Unlike the baseline which looks at one image at a time, this model looks at 16 consecutive frames simultaneously — like watching a short clip rather than a single photo. This lets it understand how the skull boundary moves and stays consistent, producing predictions that are stable across the whole video rather than flickering frame to frame.

⚙️ Technical detail

Joint design with my teammate. The 3D U-Net (365K parameters — intentionally lightweight for Kaggle's resource constraints) processes 16-frame clips with volumetric convolutions. Encoder uses 3D conv blocks with spatial-only MaxPool3D (1×2×2) to preserve the temporal dimension during downsampling. Trained with hybrid BCE + Dice loss, Adam optimiser, ReduceLROnPlateau, early stopping. Best checkpoint at epoch 19 of 30: Train Dice = 0.948, Val Dice = 0.933.

The Temporal Stability Module computes HC per frame across each predicted sequence and reports temporal standard deviation and a reliability score R = 1 / (1 + temporal_std). Sequences with unstable predictions are automatically flagged — giving clinicians a machine-generated confidence measure alongside every measurement.

Limitations & Future Work

Honest about what remains

Current limitations

The video model was trained on 400 of 999 available images due to Kaggle's 20 GB disk limit, at half resolution. Scaling to the full dataset at full resolution (768×512) is the single highest-impact next step.

The Pseudo-LDDM generates motion from physics rules rather than learned tissue mechanics. Sequences are realistic in many respects but don't fully capture the biological variability of real clinical acquisitions.

Systematic HC underestimation for larger heads (>150mm) — boundary under-segmentation at the extremes. Shape priors or stronger boundary supervision may help.

What comes next

Scale to full dataset — 999 samples at 768×512. This single change is expected to substantially close the performance gap with the published baseline.

Semi-LDDM — train a lightweight optical flow network on a small real video dataset (e.g., FPUS23) and inject learned motion vectors into our physics-constrained framework, combining data-driven learning with anatomical interpretability.

Clinical validation — a Visual Turing Test with certified sonographers to evaluate whether the synthetic sequences are indistinguishable from real clinical cine acquisitions.

★ What this project taught — uniquely

Synthetic data generation as a research contribution. When real cine-loop (video) ultrasound training data doesn't exist, you create it. The Pseudo-LDDM framework — generating physically plausible probe motion simulations from static images — was the core novel contribution and a different kind of thinking from model architecture design. Understanding how to generate training data from first principles (physics of probe motion, anatomical constraints) is directly applicable to any medical imaging domain with scarce annotations.

3D convolution for temporal consistency. This was the first project using 3D convolutional networks (3D U-Net) — where the third spatial dimension is time rather than depth. Understanding how 3D convolution captures temporal dependencies across frames, and when it adds value over applying a 2D model frame-by-frame, required building intuition about what "temporal consistency" means for a measurement task. The reliability scoring module (R=1.0 meaning every frame agreed on the same measurement) was the evaluation artifact of that reasoning.

Ellipse fitting as a post-processing step. Raw segmentation masks don't directly give circumference measurements — fitting an ellipse to the predicted mask boundary and computing the ellipse's perimeter is a geometric post-processing step that most segmentation projects don't require. This bridged computer vision (segmentation) with clinical measurement (circumference in mm), and required understanding the distinction between predicting a region and measuring a quantity from that prediction.
