Overview

Automated visual quality control using classical feature engineering

20 images, varying lighting, one discriminating signal — texture.

In manufacturing, distinguishing "good" from "bad" components by visual inspection is slow and error-prone under fatigue. This project builds an automated pipeline for a specific industrial quality control problem: photographed components on a circular platter where the key discriminating feature is surface texture — good components show clear cross/strip patterns; bad ones appear blank.

The dataset consists of 20 images photographed under varying external lighting conditions causing significant over/under-exposure, which makes texture patterns (the very thing we need to detect) difficult to see. Solving the lighting problem and then reliably detecting texture was the core technical challenge.

Key finding from preprocessing experiments: Component detection actually performed better on the original images than on the preprocessed versions. Over-normalisation by histogram equalization removed some of the intensity contrast that helped edge detection locate component boundaries. This result — that more preprocessing isn't always better — directly informed the pipeline decision to use original images for detection.

Pipeline

Full automated classification pipeline

A 4-stage pipeline from raw image to classified component, with Gabor and Wavelet feature extraction running in parallel from the same isolated component patches.

Tarun (Lead)

Stage 1 — Image Preprocessing

RGB → Grayscale → Gaussian smoothing (noise reduction) → Histogram equalization (lighting normalization). Evaluated against direct detection on originals — originals won.

Tarun (Lead)

Stage 2 — Component Detection

Sobel edge detection on original images → Adaptive thresholding (handles varying illumination) → Morphological operations (dilation, erosion, closing) to fill gaps → Connected component labeling → Bounding box generation. Saved to detected_components/.

Tarun (Lead)

Stage 3 — Component Isolation

Cropped individual components from original images using detected bounding boxes. Isolated patches saved for downstream feature extraction. Also manually annotated all 20 images using MATLAB Image Labeler (green = good, red = bad) to generate ground truth .mat files.

Jeevan + Rena

Stage 4a — Gabor Feature Extraction

Gabor filters applied at multiple orientations and spatial frequencies. Two configurations evaluated (Gabor 1 and Gabor 2 with different frequency/orientation parameters). Statistical features (mean, std) computed over filter responses per component.

Jeevan + Rena

Stage 4b — Wavelet Feature Extraction

Multi-level wavelet decomposition using Haar, Daubechies (db2), Symlet (sym4), and Coiflet (coif1) families. Multi-scale texture features captured at different frequency bands.

Ramyasri

Stage 5 — Classification

Threshold-based approach: if frequency response > threshold → Textured → Good. Mixed classifiers combining Gabor and Wavelet responses for improved robustness. Evaluated with mIoU (localization) + Accuracy/Precision/Recall/F1 (classification).

Feature Methods

Gabor filters and Wavelet transforms — two complementary views of texture

Gabor filters — orientation-selective texture

Gabor filters are band-pass filters that respond to oriented spatial frequency patterns — essentially, they detect texture at a specific angle and scale. For this quality control task, good components have directional cross/strip patterns, making Gabor filters a natural fit. Two configurations were evaluated to find the optimal orientation and frequency set for this specific texture pattern.

Per-component features: mean and standard deviation of filter responses across all orientations and frequencies.

Wavelet transforms — multi-scale texture decomposition

Wavelet transforms decompose the component image into sub-bands at different frequency scales simultaneously. Unlike Gabor filters (which probe one orientation at a time), wavelets capture the full multi-scale energy distribution of the component texture — how much energy exists at each frequency band and spatial location.

★ What this project taught — uniquely

MATLAB as a scientific computing environment for image analysis. Every other project in this portfolio uses Python with PyTorch or TensorFlow. This was the only project built entirely in MATLAB Live Scripts — learning to use the Image Processing Toolbox, Wavelet Toolbox, and MATLAB Image Labeler for ground truth annotation. The toolbox-based approach (where morphological operations, connected component labeling, and wavelet decomposition are built-in primitives) taught a different mental model for image processing than writing things from scratch in Python.

Classical feature engineering as an interpretable alternative to deep learning. All other projects in this portfolio use neural networks. This project deliberately uses handcrafted features — Gabor filter responses and wavelet energy decompositions — and a threshold classifier. This made the discriminating features completely visible: you can see exactly which frequency band or orientation a component differs on. It's a direct contrast to the black-box representations that CNNs learn, and understanding when classical feature engineering is sufficient (and interpretable) is a practically important skill.

Preprocessing isn't always better — empirical validation matters. The counterintuitive finding that histogram equalization degraded detection performance (by removing the very contrast that edge detection relied on) was a concrete reminder that preprocessing decisions should be validated empirically, not assumed to help. This lesson is transferable to every deep learning preprocessing pipeline — augmentation and normalization choices can hurt as easily as help.

Manual annotation and ground truth quality. Creating the 20 .mat annotation files using MATLAB Image Labeler — manually marking good components with green boxes and bad ones with red — made the relationship between annotation quality and evaluation metric quality tangible. The mIoU score you get is only as good as the ground truth you annotated.
